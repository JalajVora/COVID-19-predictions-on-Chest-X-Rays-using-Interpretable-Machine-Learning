Data-Set

Our dataset is a collection of 1313 CXR images containing COVID-19 positive and negative samples. There is a problem of class imbalance as we had 1000 negative samples compared to only 313 positive samples. The local binary patterns that we extracted from the CXR images have 58 dimensions. For class imbalance, we have implemented an oversampling technique that is provided by the R package â€˜ROSEâ€™, which basically means Random Over Sampling Examples. Itâ€™s a resampling function that generates synthetic balanced samples using holdout, bootstrap or cross-validation methods.

Random Forest

Random forest is one of the algorithms we tried to apply on the lbp features that was extracted from the X-ray images. The implementation was done through the â€˜randomForestâ€™ package that is available in R. The package uses Breimanâ€™s method of random forests for classification. But this package has the flexibility to be applied in a regression problem as well. In the default parameter setting, the no. of trees that the algorithm learns is 200. With that, we achieved an accuracy of about 91.69% with an F1 score of 0.71 in the original benchmark dataset. On the over-sampled data, the accuracy dropped to 78.88% with an F1 score of 0.74.

Decision Trees

For decision trees, we used the R library â€˜rpartâ€™ which is generally used to learn classification or regression trees. We ran this algorithm in itâ€™s default setting( minsplit = 20 and minbucket = round(minsplit/3) ) where minsplit is the minimum number of observations that must exist in a node for a split to occur and min bucket is the minimum number of observations in any leaf node. The accuracy that we obtained on the original benchmark dataset was 90% with an F1 score of 0.65. In the over-sampled dataset, the accuracy dropped to 81.67% with 0.81 as F1 score.
